== Overview

The `https://github.com/graphite-project/graphite-web[Graphite]` plugin connects QALIPSIS to Graphite using the pickle or plaintext protocol.

Technology addressed::

Graphite: https://graphiteapp.org/

Dependency::
`io.qalipsis:plugin-graphite`

Namespace in scenario::
`graphite()`

Client library::
* https://netty.io/[Netty]
* https://ktor.io/[Ktor]

Supported steps::
* <<Poll step,`poll`>>: polls data from Graphite. 
* <<Save step,`save`>>: saves events and records to Graphite. 

Examples of usage are provided later in this <<Examples of usage,section>>.

== Configuration

=== Sample configuration file

Graphite configuration settings are configured in a graphite.yaml file. 

[source,kotlin,linenumbers]
----
campaign:
  name: random-graphite-name
  configuration:
    validation:
      max-execution-duration: PT40S
----

== Examples of usage

== Poll step
The poll step within the the Graphite plugin collects metrics from a Graphite server and broadcasts them to other components.

Ancestor::
Scenario or Step

Functionality::  
The ‘poll’ step is typically used to collect and distribute metrics for monitoring and visualization purposes.
 
Example::
[source,kotlin,linenumbers]
----
graphite().poll {
  name = "my-poll-step"

  connect { 
    server("http://localhost:8080")
  }

  query( 
    GraphiteQuery("device.battery-state.*")
      .from( 
        GraphiteMetricsTime(
          2,
          GraphiteMetricsTimeSignUnit.MINUS,
          GraphiteMetricsTimeUnit.DAYS
        )
      )
      .noNullPoints(true)
  )
  broadcast(123, Duration.ofSeconds(20))
  pollDelay(Duration.ofSeconds(1))
}
----


Reference documentation::
Refer to the https://github.com/graphite-project/graphite-web/tree/master/docs[Graphite] documentation for parameter and configuration information. 

== Save step
The `save` step within the Graphite plugin persists records into a Graphite database using the save query.

Ancestor::
Scenario or Step

Functionality::
The `save` step's input and step context are used to generate documents to save into Graphite.

Example::
[source,kotlin,linenumbers]
----
.graphite()
.save {

  connect {
    server("localhost", 2003)
  }

  records { StepContext, input ->
    listOf(
      GraphiteRecord(
        metricPath = "device.battery-state.${input.deviceId.lowercase()}",
        value = input.batteryLevel,
        timestamp = input.timestamp
      )
    )
  }
  monitoring {
    events = false
    meters = true
  }
}
----

Reference documentation::
Refer to the https://github.com/graphite-project/graphite-web/tree/master/docs[Graphite] documentation for further parameter and configuration information. 

=== GraphiteSaveAndPoll scenario
[source,kotlin,linenumbers]
----
scenario {
  minionsCount = 20
  profile {
    immediate()
  }
}
  .start() 
  .jackson() <1>
  .csvToObject(mappingClass = BatteryState::class) {  <2>
    classpath(path = "battery-levels.csv")
    header { <3>
      column(name = "device_id")
      column(name = "timestamp")
      column(name = "battery_level").integer()
    }
    unicast()
  }
  .map { <4>
    val timeReportedToToday = LocalTime.from(it.value.timestamp.atZone(ZoneId.systemDefault()))
    val deviceTimestampReportedToToday = 
        yesterday.atTime(timeReportedToToday.hour, timeReportedToToday.minute, timeReportedToToday.second)

    it.value.copy(timestamp = deviceTimestampReportedToToday.toInstant(localZoneOffset))
  }
  .graphite() <5> 
  .save {  <6>

    connect {  <7>
      server("localhost", 2003)
    }

    records { _, input -> <8>
      listOf(
        GraphiteRecord(
          metricPath = "device.battery-state.${input.deviceId.lowercase()}",
          value = input.batteryLevel,
          timestamp = input.timestamp
        )
      )
    }

    monitoring { <9>
      events = false
      meters = true
    }

  }
  .delay(Duration.ofSeconds(1))
  .map { it.input }
  .innerJoin()
  .using { correlationRecord ->
    correlationRecord.value.deviceId
  }
  .on {
    it.graphite().poll { <10>
      name = "my-poll-step"

      connect {  <11>
        server("http://localhost:8090/render")
      }

      monitoring {
        events = false
        meters = true
      }

      query { <12>
        withTarget("device.battery-state.*")                              
          .from(GraphiteMetricsTime(-2, GraphiteMetricsTimeUnit.DAYS)) 
          .noNullPoints(true)
      }
      broadcast(123, Duration.ofSeconds(20)) <13>
      pollDelay(Duration.ofSeconds(1)) <14>
    }
      .flatten()
      .map { record -> <15>
        val dataPoint = record.dataPoints.first()
        BatteryState(
          deviceId = record.target.substringAfter("device.battery-state.").uppercase(),
          batteryLevel = dataPoint.value.toInt(),
          timestamp = dataPoint.timestamp
        )
      }
  }
  .having { correlationRecord ->
    correlationRecord.value.deviceId
  }
  .filterNotNull()
  .verify { result -> <16>
    result.asClue {
      assertSoftly {
        it.first.batteryLevel shouldBeExactly it.second.batteryLevel
      }
    }
  }

----
<1> Start the jackson step to fetch data from the csv file. 
<2> Use the csvToObject method to map csv entries to list of utils.BatteryState object.
<3> Define the header of the csv file.
<4> Normalize each input record's timestamp before saving to Graphite.
<5> Declare the Graphite plugin namespace for the following steps.
<6> Start the Save step to generate and send GraphiteRecord entries to Graphite.
<7> Configure the Graphite ingestion endpoint (host/port) used by the Save step (plaintext/pickle TCP ingest).
<8> Build the list of GraphiteRecord objects from the step input: sets metricPath, value, and timestamp that will be persisted to Graphite.
<9> Configure step monitoring behavior: disables event publication and enables meter reporting for this step.
<10> Start a Poll step inside the join: issues a Graphite query for metrics correlated to the saved records and returns polled time series.
<11> Configure the Graphite HTTP render endpoint used to execute poll queries (URL for read/render API).
<12> Define the poll query: target pattern, time window (from(...)) and noNullPoints(true) to exclude series containing only null points.
<13> Set the singleton delivery strategy to broadcast with bufferSize = 123 and idleTimeout = 20s.
<14> Specify the delay between poll attempts (throttling/wait time to allow data ingestion and to limit request frequency).
<15> Map each polled record to the domain BatteryState: extracts the first data point, parses deviceId from the metric path, converts the value to Int, and preserves the timestamp.
<16> Perform the assertion step: compares the saved batteryLevel against the polled batteryLevel.

== Configuration of publication of events and meters to Graphite.

NOTE::
Configure `event` and `meter` publication settings in the factory configuration, separate from the scenario DSL.

Refer to xref:up-and-running/configuration/provide-configuration.adoc[Provide configuration] for information about specifying the configuration to QALIPSIS.

Events::
`events.export.graphite`

Meters::
`meters.export.graphite`

* `ERROR` = confirmed problems that need attention. +

*Applicable Steps:* Events +

<2> Configure event publishing to Graphite. 
<3> Start meter configuration.
<4> Configure meter publishing to Graphite.
