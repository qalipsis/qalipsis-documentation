= TimescaleDB plugin [.badge.oss]## ##[.badge.commercial]## ##

== Overview
The TimescaleDB plugin connects QALIPSIS to https://docs.timescale.com/timescaledb/latest[TimescaleDB].

Technology addressed::
* PSQL: https://www.postgresql.org/
* TimescaleDB: https://www.timescale.com/

Dependency::
`io.qalipsis:plugin-timescaledb`

Namespace in scenario::
`timescaledb()`

Client library::
https://docs.timescale.com/keywords/Toolkit/

== Configuration of publication of events and meters to TimescaleDB.

NOTE::
Configure `event` and `meter` publication settings in the factory configuration, separate from the scenario DSL.

Refer to xref:up-and-running/configuration/provide-configuration.adoc[Provide the configuration to QALIPSIS] for information about specifying the configuration to QALIPSIS.

Events::
`events.export.timescaledb`

Meters::
`meters.export.timescaledb`

=== Event and meter parameters

[cols="1,4" options="header"]
|===
| *Parameter*
| *Description*

|`enabled`
a|Enables or disables publishing to TimescaleDB. +
*Applicable steps:* Events, Meters +
*Optional/Required:* Required +
*Data Type:* Boolean +
*Default Value:* `false` +

.Example
[source,yaml]
----
enabled: true
----

|`host`
a|Hostname or IP address of the TimescaleDB/PostgreSQL server. Cannot be blank. +
*Applicable steps:* Events, Meters +
*Optional/Required:* Required +
*Data Type:* String +
*Default Value:* `localhost` +

.Example
[source,yaml]
----
host: localhost
----

|`port`
a|Port number on which the TimescaleDB/PostgreSQL server listens. +
*Applicable steps:* Events, Meters +
*Optional/Required:* Required +
*Data Type:* Positive integer +
*Default Value:* `5432` +

.Example
[source,yaml]
----
port: 5432
----

|`database`
a|Name of the TimescaleDB database. Cannot be blank. +
*Applicable steps:* Events, Meters +
*Optional/Required:* Required +
*Data Type:* String +
*Default Value:* `qalipsis` +

.Example
[source,yaml]
----
database: qalipsis
----

|`schema`
a|Name of the schema where events or meters are stored. Cannot be blank. +
*Applicable steps:* Events, Meters +
*Optional/Required:* Required +
*Data Type:* String +
*Default Value:* +
- Events: `meters` +
- Meters: `meters` +

.Example
[source,yaml]
----
schema: meters
----

|`min-level`
a|Minimum event level to publish. Only applicable to Events. Acceptable values are `TRACE`, `DEBUG`, `INFO`, `WARN`, `ERROR`, `OFF`. +
* `TRACE` = most verbose; shows internal and fine-grained details for deep troubleshooting. +
* `DEBUG` = diagnostic information useful for investigating issues. +
* `INFO` = routine operational messages and high-level progress. +
* `WARN` = indications of potential problems that do not stop processing. +
* `ERROR` = confirmed problems that require attention. +
* `OFF` = disables event publishing. +
*Applicable steps:* Events +
*Optional/Required:* Required +
*Data Type:* ENUM +
*Default Value:* `INFO` +

.Example
[source,yaml]
----
min-level: INFO
----

|`linger-period`
a|Maximum time to buffer events before forcing a flush to TimescaleDB, even if the batch size is not reached. Only applicable to Events. +
*Applicable steps:* Events +
*Optional/Required:* Optional +
*Data Type:* Duration +
*Default Value:* `10S` +

.Example
[source,yaml]
----
linger-period: 10s
----


|`batch-size`
a|Maximum number of events buffered between two publications to TimescaleDB. Only applicable to Events. +
*Applicable steps:* Events +
*Optional/Required:* Required +
*Data Type:* Positive integer +
*Default Value:* `20000` +

.Example
[source,yaml]
----
batch-size: 20000
----

|`publishers`
a|Number of concurrent publishers that can run. Only applicable to Events. Must be at least 1. +
*Applicable steps:* Events +
*Optional/Required:* Required +
*Data Type:* Positive integer +
*Default Value:* `1` +

.Example
[source,yaml]
----
publishers: 1
----

|`username`
a|Username for basic authentication when connecting to TimescaleDB. +
*Applicable steps:* Events, Meters +
*Optional/Required:* Required +
*Data Type:* String +
*Default Value:* None +

.Example
[source,yaml]
----
username: qalipsis_user
----

|`password`
a|Password for basic authentication when connecting to TimescaleDB. +
*Applicable steps:* Events, Meters +
*Optional/Required:* Required +
*Data Type:* String +
*Default Value:* None +

.Example
[source,yaml]
----
password: qalipsis_pwd
----

|`enable-ssl`
a|Enables or disables SSL/TLS encryption for the database connection. +
*Applicable steps:* Events, Meters +
*Optional/Required:* Optional +
*Data Type:* Boolean +
*Default Value:* `false` +

.Example
[source,yaml]
----
enable-ssl: false
----

|`ssl-mode`
a|Defines the SSL mode for the database connection. Acceptable values are `DISABLE`, `ALLOW`, `PREFER`, `REQUIRE`, `VERIFY_CA`, `VERIFY_FULL`. +
*Applicable steps:* Events, Meters +
*Optional/Required:* Optional +
*Data Type:* ENUM +
*Default Value:* `PREFER` +

.Example
[source,yaml]
----
ssl-mode: PREFER
----

|`ssl-root-cert`
a|Path to the SSL root certificate file for use in SSL/TLS connections. Optional. +
*Applicable steps:* Events, Meters +
*Optional/Required:* Optional +
*Data Type:* String +
*Default Value:* None +

.Example
[source,yaml]
----
ssl-root-cert: /path/to/root-cert.pem
----

|`ssl-cert`
a|Path to the SSL client certificate file for use in SSL/TLS connections. Optional. +
*Applicable steps:* Events, Meters +
*Optional/Required:* Optional +
*Data Type:* String +
*Default Value:* None +

.Example
[source,yaml]
----
ssl-cert: /path/to/client-cert.pem
----

|`ssl-key`
a|Path to the SSL client private key file for use in SSL/TLS connections. Optional. +
*Applicable steps:* Events, Meters +
*Optional/Required:* Optional +
*Data Type:* String +
*Default Value:* None +

.Example
[source,yaml]
----
ssl-key: /path/to/client-key.pem
----

|`prefix`
a|Prefix to prepend to metric names when publishing to TimescaleDB. Only applicable to Meters. +
*Applicable steps:* Meters +
*Optional/Required:* Optional +
*Data Type:* String +
*Default Value:* `qalipsis` +

.Example
[source,yaml]
----
prefix: qalipsis
----

|`min-idle-connection`
a|Minimum number of idle database connections to maintain in the connection pool. Only applicable to Meters. Must be at least 1. +
*Applicable steps:* Meters +
*Optional/Required:* Optional +
*Data Type:* Positive integer +
*Default Value:* `1` +

.Example
[source,yaml]
----
min-idle-connection: 1
----

|`max-pool-size`
a|Maximum number of database connections in the connection pool. Only applicable to Meters. Must be at least 1. +
*Applicable steps:* Meters +
*Optional/Required:* Optional +
*Data Type:* Positive integer +
*Default Value:* `1` +

.Example
[source,yaml]
----
max-pool-size: 1
----

|`init-schema`
a|Enables automatic creation or update of the schema for meters at startup. Only applicable to Meters. +
*Applicable steps:* Meters +
*Optional/Required:* Optional +
*Data Type:* Boolean +
*Default Value:* `true` +

.Example
[source,yaml]
----
init-schema: true
----

|===

=== Example configuration file
[source,yaml,linenumbers]
----
events: <1>
  export:
    timescaledb: <2>
      enabled: true
      min-level: INFO
      host: localhost
      port: 5432
      database: qalipsis
      schema: meters
      linger-period: 10s
      batch-size: 20000
      publishers: 1
      username: qalipsis_user
      password: qalipsis_pwd
      enable-ssl: false
      ssl-mode: PREFER

meters: <3>
  export:
    timescaledb: <4>
      enabled: true
      host: localhost
      port: 5432
      database: qalipsis
      schema: meters
      username: qalipsis_user
      password: qalipsis_pwd
      enable-ssl: false
      ssl-mode: PREFER
----

<1> Start event publishing configuration.
<2> Configure event publishing to TimescaleDB.
<3> Start meter publishing configuration.
<4> Configure meter publishing to TimescaleDB.

//-
Tip::
TimescaleDB requires valid credentials for authentication. Ensure that the username and password for a properly configured TimescaleDB user are provided in the configuration.
