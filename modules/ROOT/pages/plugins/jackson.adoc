== Overview
The Jackson plugin supports reading a (https://github.com/FasterXML/jackson[JSON, XML or CSV]) file and converting the data to an object.

Technologies addressed::
https://github.com/FasterXML/jackson[Class JsonMapper]

Dependency::
`io.qalipsis.plugin:qalipsis-plugin-jackson`

Namespace in scenario::
`jackson()`

Client library::
JsonMapper: refer to http://fasterxml.github.io/jackson-databind/javadoc/2.11/com/fasterxml/jackson/databind/json/JsonMapper.html[FasterXML].

Supported steps::
* <<csvToMap and csvToObject, csvToMap and csvToObject>>: reads a CSV file and converts the data to a Map or an Object.
* <<jsonToMap and jsonToObject, jsonToMap and jsonToObject>>: reads a JSON file and converts the data to a Map or an Object.
* <<xmlToObject, xmlToObject>>: reads an XML file and converts the data to an Object.

Examples of usage are provided later in this <<Examples of usage,section>>.

== Configuration parameters
Available parameters are described in the table below.
[cols="1,4" options="header"]
|===
| *Parameter*
| *Description*

|`classpath(file)`
a| Reads the data from a specified class path resource; `file` is the path to the resource with the leading slash ignored. +
*Applicable Steps:* csvToMap, csvToObject, csvToList, jsonToMap, jsonToObject, xmlToObject +
*Optional/Required:* Optional +
*Data Type:* String +
*Default Value:* N/A +

.Example
[source,kotlin,linenumbers]
----
classpath("battery-levels.csv")
----

|`file (path)`
a|Reads the data from a specified plain file in the file system; `path` is the path to the file, either absolute or relative to the current working directory. +
*Applicable Steps:* csvToMap, csvToObject, csvToList, jsonToMap, jsonToObject, xmlToObject +
*Optional/Required:* Optional +
*Data Type:* String +
*Default Value:* N/A +

.Example
[source,kotlin,linenumbers]
----
file("/battery-levels.csv")
----

|`url(url)`
a| Reads the data from a specified remote web resource. +
*Applicable Steps:* csvToMap, csvToObject, csvToList, jsonToMap, jsonToObject, xmlToObject +
*Optional/Required:* Optional +
*Data Type:* url +
*Default Value:* N/A +

.Example
[source,kotlin,linenumbers]
----
url("https://raw.githubusercontent.com/iot-data/sensors/main/battery-levels.csv")
----

|`encoding(charset)`
a| Sets the character encoding used to open the `InputStreamReader` for the source file. Accepts either the charset name as a `String` (passed to `Charset.forName()`) or a `java.nio.charset.Charset` instance directly. +
*Applicable Steps:* csvToMap, csvToObject, csvToList, jsonToMap, jsonToObject, xmlToObject +
*Optional/Required:* Optional +
*Data Type:* `String` or `java.nio.charset.Charset` +
*Default Value:* `UTF-8` +

.Example
[source,kotlin,linenumbers]
----
encoding("windows-1252") or encoding(Charsets.UTF_8)
----

|`columnSeparator(sep)`
a| Sets the character used to split columns in a CSV row. +
*Applicable Steps:* csvToMap, csvToObject, csvToList +
*Optional/Required:* Optional +
*Data Type:* `Char` +
*Default Value:* `,` +

.Example
[source,kotlin,linenumbers]
----
columnSeparator(';')
----

|`quoteChar(quoteChar)`
a| Sets the character used to quote cell values that contain the column separator or line separator, preventing them from being split. +
*Applicable Steps:* csvToMap, csvToObject, csvToList +
*Optional/Required:* Optional +
*Data Type:* `Char` +
*Default Value:* `"` +

.Example
[source,kotlin,linenumbers]
----
quoteChar('"')
----

|`escapeChar(escapeChar)`
a| Sets the character used to escape special characters inside a cell value so they are not interpreted as delimiters. +
*Applicable Steps:* csvToMap, csvToObject, csvToList +
*Optional/Required:* Optional +
*Data Type:* `Char` +
*Default Value:* `\` +

.Example
[source,kotlin,linenumbers]
----
escapeChar('\\')
----

|`lineSeparator(sep)`
a| Sets the line separator used to split rows. Accepts either a `String` or a `Char`. An empty string throws `InvalidSpecificationException`. +
*Applicable Steps:* csvToMap, csvToObject, csvToList +
*Optional/Required:* Optional +
*Data Type:* `String` or `Char` +
*Default Value:* `System.lineSeparator()` +

.Example
[source,kotlin,linenumbers]
----
lineSeparator('\n') or lineSeparator("\r\n")
----

|`allowComments()`
a| When called, any line whose first non-whitespace character is `#` is silently skipped by the CSV parser. No argument is taken; calling the function sets the flag to `true`. +
*Applicable Steps:* csvToMap, csvToObject, csvToList +
*Optional/Required:* Optional +
*Data Type:* Function (no arguments) +
*Default Value:* N/A +

.Example
[source,kotlin,linenumbers]
----
allowComments()
----

|`header { ... }`
a| Configures the column structure of the CSV file. The lambda receiver is `CsvHeaderConfiguration`, which exposes `withHeader()`, `skipFirstDataRow()`, and `column(...)`. Parsing options (`columnSeparator`, `quoteChar`, `escapeChar`, `allowComments`) must be set at the step level, not inside this block. +
*Applicable Steps:* csvToMap, csvToObject, csvToList +
*Optional/Required:* Optional +
*Data Type:* `CsvHeaderConfiguration.() -> Unit` +
*Default Value:* N/A +

.Example
[source,kotlin,linenumbers]
----
header {
   withHeader()
   column("deviceId").string()
   column("batteryLevel").integer()
   column("timestamp").long()
}
----

|`header.withHeader()`
a| Declares that the first non-skipped line of the CSV file is the header row. Its cell values become the keys of the output map (for `csvToMap`) or the property names used for binding (for `csvToObject`). Must be called inside `header {}`. +
*Applicable Steps:* csvToMap, csvToObject, csvToList +
*Optional/Required:* Optional +
*Data Type:* Function (no arguments) +
*Default Value:* N/A +

.Example
[source,kotlin,linenumbers]
----
header {
   withHeader()
}
----

|`header.skipFirstDataRow()`
a| Skips the first data row after the header. Use this when an external tool adds a units row or summary row immediately after the header line. Must be called inside `header {}`. +
*Applicable Steps:* csvToMap, csvToObject, csvToList +
*Optional/Required:* Optional +
*Data Type:* Function (no arguments) +
*Default Value:* N/A +

.Example
[source,kotlin,linenumbers]
----
header {
   skipFirstDataRow()
}
----

|`column(name)` / `column(index, name)` / `column(index)` (inside `header {}`)
a| Registers a column by name (appended in declaration order) or by explicit 0-based index and name. Returns a `CsvColumnConfiguration` on which type methods (`string()`, `integer()`, `long()`, `double()`, `boolean()`, etc., and their nullable variants) can be chained. Without a type call, the default type is `NULLABLE_STRING`. `array(separator)` may be chained after a type call to split the cell on an intra-cell separator. `ignoreError(default)` may be chained to suppress conversion errors and substitute a fallback value. +
*Applicable Steps:* csvToMap, csvToObject, csvToList +
*Optional/Required:* Optional +
*Data Type:* Function that accepts a `String` name, an `Int` index alone, or an `Int` index and a `String` name, and returns a `CsvColumnConfiguration` for chaining type and parsing options. +
*Default Value:* N/A +

.Example
[source,kotlin,linenumbers]
----
column("voltage").double()
column("tags").string().array(",")
column("notes").string(true).ignoreError("")
----

|`mapper { ... }`
a| Tweaks the `JsonMapper` (JSON steps) or `XmlMapper` (XML step) used to deserialize records. The lambda receives the already-configured mapper instance (with `KotlinModule`, `JavaTimeModule`, `Jdk8Module`, and `BeanIntrospectionModule` pre-registered) and adds behaviour on top. This block does not replace the default module registrations. +
*Applicable Steps:* jsonToMap, jsonToObject, xmlToObject +
*Optional/Required:* Optional +
*Data Type:* `(JsonMapper) -> Unit` for JSON steps; `(XmlMapper) -> Unit` for the XML step. +
*Default Value:* No-op lambda `{}` +

.Example (JSON)
[source,kotlin,linenumbers]
----
mapper { jsonMapper ->
    jsonMapper.configure(
        com.fasterxml.jackson.databind.DeserializationFeature.FAIL_ON_UNKNOWN_PROPERTIES, false
    )
}
----

.Example (XML)
[source,kotlin,linenumbers]
----
mapper { xmlMapper ->
    xmlMapper.configure(
        com.fasterxml.jackson.databind.DeserializationFeature.FAIL_ON_UNKNOWN_PROPERTIES, false
    )
}
----

== Examples of usage

=== csvToMap and csvToObject
The `csvToMap` step reads a CSV resource (file, classpath resource, or URL) and returns each row as a `Map` keyed by column name. The `csvToObject` step reads the same resource and deserializes each row directly into a typed object.

Ancestor::
Step

Functionality::
The `csvToMap` step converts each CSV row to a `Map` and pushes it to the next step. +
The `csvToObject` step maps each CSV row to an instance of the specified target class and pushes it to the next step.


Example::
+
[source,kotlin,linenumbers]
----
// Example 1: csvToMap
jackson().csvToMap {
   classpath("battery-levels.csv")
   escapeChar('E')
   quoteChar('"')
   columnSeparator(';')
   withHeader()
   allowComments()
   header {
      column("1st value").double()
      column("2th value").string().array(",")
      column("3rd value").string(true)
      column("4th value").string(true)
   }
}

// Example 2: csvToObject
jackson()
.csvToObject(BatteryState::class) {
   classpath("battery-levels.csv")
   header {
      column("deviceId")
      column("timestamp")
      column("batteryLevel").integer()
   }
   unicast()
}
----

=== csvToList
The `csvToList` step within the Jackson plugin reads a CSV resource (classpath, file, or url) and returns each item as a `List` of values, preserving column order. Use `csvToList` when a positional list of values per row is preferred over a `Map` or a typed object.

Ancestor::
Step

Functionality::
The `csvToList` step converts each CSV row to a `List<Any>` (or `List<String>` when no typed header is configured) and pushes that list to the next step. A `header` block may be provided to define column types and parsing behavior; columns are emitted in the configured order.

Examples::
+
[source,kotlin,linenumbers]
----
//Each row is emitted as List<String> (no header parsing).
jackson().csvToList {
   classpath("battery-levels.csv")
}

//Each row emitted as List<Any> with parsed types in the column order.
jackson().csvToList {
   file("/data/measurements.csv")
   header {
      withHeader()            //If file contains header row.
      column("deviceId").string()
      column("batteryLevel").integer()
      column("timestamp").long()
   }
   broadcast()
}
----

Tip::
Only one path (`classpath(path)`, `file(path)`, or `url(url)`) to the resource is allowed for each `csvToObject` or `csvToMap` step.

Reference documentation::
Refer to the https://github.com/FasterXML/jackson[jackson] documentation for further configuration and parameter information.

=== xmlToObject
The `xmlToObject` step within the Jackson plugin reads an XML resource (classpath, file, or url) and returns each item as an instance of an object of the specified class.

Ancestor::
Step

Functionality::
The `xmlToObject` step specification has a `targetClass` property to which the output lines should be mapped.
The step reads an XML file, converts the data to an `Object`, and pushes the `Object` to the next step.

Example::
+
[source,kotlin,linenumbers]
----
jackson().xmlToObject(PojoForTest::class) {
   classpath("data.xml")
}
----

Tip::
Only one path (`classpath(path)`, `file(path)`, or `url(url)`) to the resource is allowed for each `xmlToObject` step.

Reference documentation::
Refer to the https://github.com/FasterXML/jackson[jackson] documentation for further configuration and parameter information.

=== jsonToMap and jsonToObject
The `jsonToMap` step reads a JSON resource (file, classpath resource, or URL) and returns each element as a `Map`. The `jsonToObject` step reads the same resource and deserializes each element directly into a typed object.

Ancestor::
Step

Functionality::
The `jsonToMap` step converts each JSON element to a `Map` and pushes it to the next step. +
The `jsonToObject` step maps each JSON element to an instance of the specified target class and pushes it to the next step.

Example::
+
[source,kotlin,linenumbers]
----
jackson().jsonToMap() {
   classpath("data.json")
}
jackson().jsonToObject(PojoForTest::class) {
   classpath("data.json")
}
----

Tip::
Only one path (`classpath(path)`, `file(path)`, or `url(url)`) to the resource is allowed for each `jsonToObject` or `jsonToMap` step.

Reference documentation::
Refer to the https://github.com/FasterXML/jackson[jackson] documentation for further configuration and parameter information.

=== JacksonCsvToMap scenario
Read a CSV file from the classpath and stream each row as a Map

[source,kotlin,linenumbers]
----
@Scenario("jackson-csv-to-map")
fun jacksonCsvToMap() {
    scenario {
        minionsCount = 10
        profile {
            immediate()
        }
    }
    .start()
    .jackson() // <1>
    .csvToMap { // <2>
        classpath("sensors/battery-levels.csv") // <3>
        columnSeparator(';') // <4>
        quoteChar('"')
        escapeChar('\\')
        allowComments() // <5>
        header {
            withHeader() // <6>
            column("deviceId").string() // <7>
            column("timestamp").long()
            column("batteryLevel").integer()
        }
        broadcast() // <8>
    }
}
----
<1> Call `jackson()` to enter the Jackson DSL namespace. This must be called before any Jackson step.
<2> Use `csvToMap` to read each CSV row and emit it as a `Map` keyed by the column names defined in `header {}`.
<3> Specify the path to the CSV file on the JVM classpath. A leading `/` is automatically ignored.
<4> Set the column separator to `;`. Define `quoteChar` and `escapeChar` to match the format of your CSV file.
<5> Call `allowComments()` to skip any lines that begin with `#`.
<6> Call `withHeader()` to indicate that the first row of the file contains the column names.
<7> Define each column and its data type. Available types include `string()`, `long()`, and `integer()`.
<8> Call `broadcast()` to send every record to all minions.
---

=== JacksonCsvToObject scenario
Read a CSV file and deserialize each row directly to a typed POJO

[source,kotlin,linenumbers]
----
@Scenario("jackson-csv-to-object")
fun jacksonCsvToObject() {
    scenario {
        minionsCount = 20
        profile {
            immediate()
        }
    }
    .start()
    .jackson() //<1>
    .csvToObject(mappingClass = BatteryState::class) { // <2>
        classpath("battery-levels.csv")
        columnSeparator(';')
        quoteChar('"')
        escapeChar('\\')
        allowComments()
        header {
            skipFirstDataRow() // <3>
            column("deviceId").string()
            column("timestamp").long()
            column("batteryLevel").integer()
        }
        unicast() // <4>
    }
    .map { record -> record.value } // <5>
}
----
<1> Call `jackson()` to enter the Jackson DSL namespace. This must be called before any Jackson step.
<2> Specify the target class to map each CSV row to. The step emits one typed object per row to the next step.
<3> Call `skipFirstDataRow()` to skip the first data row after the header line.
<4> Call `unicast()` to deliver each record to exactly one minion.
<5> Transform the output of the CSV reader to a plain `BatteryState` object.
---

=== JacksonCsvToList scenario
Read a CSV file into a List of values using index-based column mapping

[source,kotlin,linenumbers]
----
@Scenario("jackson-csv-to-list")
fun jacksonCsvToList() {
    scenario {
         minionsCount = 5
         profile {
             immediate()
         }
     }
    .start()
    .jackson()  //<1>
    .csvToList { // <2>
        classpath("exports/raw-readings.csv")
        columnSeparator(';')
        quoteChar('"')
        escapeChar('\\')
        allowComments()
        header {
            column(0, "reading").double() // <3>
            column(1, "labels").string().array(",") // <4>
            column(2, "description").string(true) // <5>
            column(3, "notes").string(true).ignoreError("") // <6>
        }
        broadcast()
    }
    .map { record -> record.value }
}
----
<1> Call `jackson()` to enter the Jackson DSL namespace. This must be called before any Jackson step.
<2> Use `csvToList` to read each CSV row and emit it as an ordered `List` of values to the next step.
<3> Define the first column by index and name, and set its type to `double()`.
<4> Call `array(",")` to split the cell value on `","` and emit the result as a `List<String>`.
<5> Pass `true` to `string()` to trim leading and trailing whitespace from the cell value.
<6> Call `ignoreError("")` to substitute `""` for any column that fails to parse, instead of failing the entire row.
---

=== JacksonJsonToObject scenario
Read a JSON array and deserialize each element to a typed POJO with a custom mapper

[source,kotlin,linenumbers]
----
@Scenario("jackson-json-to-object")
fun jacksonJsonToObject() {
    scenario {
        minionsCount = 15
        profile {
            immediate()
        }
    }
    .start()
    .jackson() //<1>
    .jsonToObject(mappingClass = SensorEvent::class) { // <2>
        url("https://raw.githubusercontent.com/iot-data/sensors/main/events.json") // <3>
        mapper { jsonMapper -> // <4>
            jsonMapper.configure(
                com.fasterxml.jackson.databind.DeserializationFeature.FAIL_ON_UNKNOWN_PROPERTIES,  //<5>
                false
            )
            jsonMapper.configure(
                com.fasterxml.jackson.databind.DeserializationFeature.READ_DATE_TIMESTAMPS_AS_NANOSECONDS, //<6>
                false
            )
        }
        broadcast() // <7>
    }
    .map { record -> record.value } // <8>
}
----
<1> Call `jackson()` to enter the Jackson DSL namespace. This must be called before any Jackson step.
<2> Specify the target class to map each JSON element to. The step emits one typed object per element to the next step.
<3> Specify the URL of the remote JSON file. The file must contain a valid top-level JSON array.
<4> Configure the JSON mapper to control how records are deserialized. Use this block to enable or disable specific Jackson features.
<5> Disable failure on unknown JSON properties. Any fields in the JSON that do not map to the target class are silently ignored.
<6> Disable reading date timestamps as nanoseconds. This allows timestamps to be read as milliseconds since the epoch.
<7> Call `broadcast()` to send every record to all minions.
<8> Transform the output of the JSON reader to a plain `SensorEvent` object for use in the next step.

=== JacksonXmlToObject scenario
Read an XML file and deserialize each element to a typed POJO.

[source,kotlin,linenumbers]
----
@Scenario("jackson-xml-to-object")
fun jacksonXmlToObject() {
    scenario {
        minionsCount = 8
        profile {
            immediate()
        }
    }
    .start()
    .jackson() //<1>
    .xmlToObject(mappingClass = Employee::class) { // <2>
        classpath("hr/employees.xml") // <3>
        mapper { xmlMapper -> // <4>
            xmlMapper.configure(
                com.fasterxml.jackson.databind.DeserializationFeature.FAIL_ON_UNKNOWN_PROPERTIES, // <5>
                false
            )
        }
        broadcast() // <6>
    }
    .map { record -> record.value } // <7>
}
----
<1> Call `jackson()` to enter the Jackson DSL namespace. This must be called before any Jackson step.
<2> Specify the target class to map each XML element to. The step emits one typed object per element to the next step.
<3> Specify the path to the XML file on the JVM classpath.
<4> Configure the XML mapper to control how records are deserialized. Use this block to enable or disable specific Jackson features.
<5> Disable failure on unknown XML properties. Any fields in the XML that do not map to the target class are silently ignored.
<6> Call `broadcast()` to send every record to all minions. Use `unicast()` instead to distribute records one per minion.
<7> Transform the output of the XML reader to a plain `Employee` object for use in the next step.
