== Overview
The Apache HTTP plugin connects QALIPSIS to HTTP and HTTPS systems using Apache HttpComponents Async Client.

Technology addressed::
HTTP/HTTPS over Apache HttpComponents: https://hc.apache.org/httpcomponents-client-5.5.x/index.html[]

Dependency::
`io.qalipsis.plugin:qalipsis-plugin-http`

Namespace in scenario::
`httpApache()`

Client library::
Apache HttpComponents Client 5 (async): refer to https://hc.apache.org/httpcomponents-client-5.5.x/index.html[Apache HttpComponents Client].

Supported steps::
* <<HTTP step,`http`>>: executes one HTTP request per received input and emits an `HttpResponse<B>`.

Examples of usage are provided later in this <<Examples of usage,section>>.

== Configuration

=== Example configuration file

Apache HTTP configuration settings are configured in scenario DSL.
[source,yaml,linenumbers]
----
campaign:
  name: apache-http-${random.shortuuid}
  configuration:
    validation:
      max-execution-duration: PT40S

events:
  root: WARN

report:
  export:
    console-live:
      enabled: false
----

| `connect.connectionStrategy.strategyType`
a| Chooses the connection strategy implementation. +
    `ON_DEMAND`: Creates a new HTTP connection on demand for each minion and closes connection after request. Use when request isolation is required, or when the target server does not support persistent connections. +

*Applicable Steps:* HTTP, defaults +
*Optional/Required:* Optional +
*Data Type:* `ConnectionStrategyType` +
*Default Value:* `ON_DEMAND` +

.Example
[source,kotlin]

        strategyType = ConnectionStrategyType.POOL

}

== Examples of usage

=== HTTP step
The `http` step within the Apache HTTP plugin executes one HTTP request per input and emits an `HttpResponse<B>`.

Ancestor::
Scenario, Step

Functionality::
The `http` step receives an input, builds a request through `request { ... }`, executes it using the configured connection strategy, and forwards the response downstream.

Example::
+
[source,kotlin,linenumbers]
----
scenario { <1>
    minionsCount = 20
}.start()
    .httpApache() <2>
    .http {
        name = "health-check" <3>
        connect { <4>
            url("http://localhost:8080")
            connectionStrategy { <5>
                strategyType = ConnectionStrategyType.ON_DEMAND
            }
        }
        request { stepContext, input -> <6>
            simple(HttpMethod.GET, "/actuator/health")
                .addHeader("Accept", "application/json") <7>
        }
    }
    .verify { response -> <8>
        assertThat(response.code).isEqualTo(200)
    }
----
<1> Creates a scenario with 20 minions.
<2> Enters the Apache HTTP plugin namespace.
<3> Names the step for reporting and logs.
<4> Defines target server base URL.
<5> Uses one client per execution (`ON_DEMAND`).
<6> Builds a GET request for each execution.
<7> Adds an `Accept` header.
<8> Verifies HTTP response status.

//-
Tips::
* Use `.deserialize(MyType::class)` when responses are JSON or XML and should be converted to typed objects.

//-
Reference Documentation::
Refer to the https://hc.apache.org/httpcomponents-client-5.5.x/index.html[Apache HttpComponents Client] documentation for further parameter and configuration information.

=== HttpFormAndJson scenario
[source,kotlin,linenumbers]
----
@Scenario("http-form-and-json") // <1>
fun httpFormAndJson() {
    scenario { // <2>
        minionsCount = 20
        profile {
            regular(100, 20) // <3>
        }
    }
    .start()
    .httpApache()
    .http {
        name = "authenticate" // <4>
        connect { // <5>
            url("https://api.example.com")
        }
        request { stepContext, input -> // <6>
            form(HttpMethod.POST, "/oauth/token")
                .addFormField("grant_type", "client_credentials")
                .addFormField("client_id", "load-test")
                .addFormField("client_secret", "load-test-secret")
        }
    }
    .deserialize(AuthToken::class) // <7>
    .map { response -> response.body!!.access_token } // <8>
    .httpApache() // <9>
    .http {
        name = "list-inventory" // <10>
        connect { // <11>
            url("https://api.example.com")
        }
        request { stepContext, input -> // <12>
            simple(HttpMethod.GET, "/inventory")
                .addHeader("Authorization", "Bearer $token") // <13>
                .addHeader("Accept", "application/json")
        }
    }
    .deserialize(Array<InventoryItem>::class) // <14>
    .verify { response -> // <15>
        assertThat(response.code).isEqualTo(200)
        assertThat(response.headers["Content-Type"] ?: "").contains("application/json")
        assertThat(response.body?.isNotEmpty()).isEqualTo(true)
    }
}
----
<1> Declare the scenario name.
<2> Enter the scenario DSL block to configure minions and the execution profile.
<3> Use a `regular` execution profile that injects 100 minions per second until all 20 are active.
<4> Name the first HTTP step. This name appears in reports and logs.
<5> Set the base URL for the authentication request.
<6> Build an `application/x-www-form-urlencoded` POST request with OAuth2 client credentials.
<7> Deserialize the authentication response body into an `AuthToken` object.
<8> Extract the bearer token string from the response and forward it as the input to the next step.
<9> Re-enter the Apache HTTP plugin namespace to start a second HTTP step. This is required each time a new `http {}` step is chained, because the output of the previous `.map {}` is a plain value, not an HTTP context.
<10> Name the second HTTP step. This name appears in reports and logs.
