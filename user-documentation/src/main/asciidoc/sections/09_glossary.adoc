== Glossary
[cols="1,2"]

|===

|  Term  |  Definition    
|Head
|Node of a Qalipsis cluster in charge of maintaining the state f the cluster, orchestrating th efactories and the campains.

|Factory  
|Node of a QUALIPSI cluster in charge of executing the scenario to generate load to a target system and verify its behavior.

|Minion
|AUnique virtual simulated user or device.

|Step  
|Atomic action that a minion can execute in a scenario

|Scenario
|Sequence of actions to be executed by minions or to poll data from a remote system.

|Directed Acyclic Graph (DAG)
|A subpart of a scenario made of a single continuous sequence of steps, having in commom to be executed in the same location bt the same minions. 

|Root DAG
|A DAG starting immediately under a scenario

|Frontend DAG
|A DAG whose followers execute on a different factory.

|Backend DAG
|A DAG that continues the execution workflow of a DAG hosted in a different factory.

|Specification 
|The configuration of a step or a scenario that describes the actions to perform and the sequence in which they are to be performed.
 
|Node
|Software entity of a QALIPSIS cluster (head or factory).

|Directive
|A single instruction published by a head to trigger an action in the factories


|Feedback
|The factory reply to a directive; the feedack provides staus of the directive's progression andor the result of the expected action.
   

|Monitoring/Time-series data
|A set of records generated by the execution of a campaign that permit the analysis of the performance of the target system and the generation of dashboards

|Meter
|A value aggregating step meters of the performance over a ahort period of tme  across all th eminions.

|Step meter
|A unique value representing the performance of a unique execution of a step by a minion (e.g., time to response, volume of data saved...)


|Event
|A timestamped snapshot of a statement executed in a step, attached to an information??? A step generally generates several events.

|Runner
|A component in charge of triggering the execution of a step onto a minion.

The  Kotlin job??? of step executions  are attached to the minios to maintain its active state.

|Selector
|The Key/Value pair attached to a step to locate its excution in a factory owning the same pair.

|Result
|The record returned by a step when executed by a minion.
The result conatins values (e.g., records fetched from a database or a response to a request) and the step meters of the execution.
The result is dispatched as output of the step context to the followers.

|Batch
|A collection of values, as opposed to a single value.

|Campaign
|A single execution of a collection of scenarios, starting at a given time and running for a given period.

|Step Report
|A consolidated view of the executions of a step.

|Campaign report
|A consolidated view of the step reports.

|Error Processing Step
|A step that is able to execute a step context?? even if exhausted.

|Proxy step
|A step acting as a middle-man to forward the step execution to a follower, using convenient strategies.

|Execution step context
|A set of fields and values representing the single execution of a single minion on a single step.

|Exhausted step context
|After an execution failure, a step context is marked as exhausted and this state is propagated to its to its descendents???.

|Tail step context
|A step context that represents the last one in the execution workflow of a single minion.

|Discarded step context
|A step execution that does not generate an output or a failure.

|DAG Under load
|A DAG that is executed by the number of minions set on the scenario. 

|Singleton DAG
|A DAG that is executed only once and supports infrastructure or optimization operations.

|Input
|The value set in a step context that is used by a step execution.

|Output
|The value (result) returned by a step execution into the step context.

|Target System
|An IT system that QALIPSIS injects load into in order to verify its ability to scale and support a high volume of users or data.
