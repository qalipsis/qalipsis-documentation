=== Assertion???
An Assertion makes it easy to perform cross-checks of  data  and  durations from different sources. Assertions allow Validation of data persistence, message exchanges or requests to remote systems. 

There are several types of assertions, the most common being:
*  A precondition assertion that is placed at the beginning of a section of code and determines the set of states under which the programmer expects the code to execute.

* A postcondition assertion tht is placed at the end of a section of code and describes the expected state at the end of execution.

=== Minion
A minion is a simulations of a website user, IoT device, or other system. Each minion performs a complete scenario.   Any number of minions can act on a  scenario.  

==== Minion Configuration parameters

=== Node
A node is a redistribution point or a communication endpoint within a QALIPSIS cluster. QALIPSIS nodes are designated as either a Head node or a Factory node.

* Head Node: The node of a QALIPSIS cluster that is responsible for orchestration and reporting.

* Factory Node: The nodes of a QALIPSIS cluster that hosts the minions to execute the load tests and assertions.
